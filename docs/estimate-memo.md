# 工程別見積メモ

## 1. 前提

- `SPA + Supabase` 構成
- 会員種別は 1 種類
- ファイルアップロード機能なし
- 管理画面なし
- 基本機能は認証、プロフィール、投稿 CRUD
- 1 人開発を想定

## 2. 想定スコープ

- 会員登録
- ログイン、ログアウト
- パスワード再設定
- プロフィール参照、更新
- 投稿一覧、投稿詳細
- 投稿作成、更新、削除
- 自分の投稿一覧
- 公開状態変更

## 3. ざっくりした段取り

1. 基本設計の確定
2. 詳細設計
3. 開発環境構築
4. Supabase 実装
5. フロント実装
6. テスト、修正
7. リリース準備

## 4. 工程別の目安

| 工程 | 内容 | 目安 |
|---|---|---|
| 基本設計の確定 | 基本設計の確認、修正反映、スコープ確定 | 1〜2 営業日 |
| 詳細設計 | 画面項目、テーブル詳細、RLS 方針の整理 | 2〜4 営業日 |
| 開発環境構築 | SPA 雛形、Supabase 接続、ルーティング初期設定 | 1〜2 営業日 |
| Supabase 実装 | テーブル作成、Migration、Auth、RLS 設定 | 2〜4 営業日 |
| フロント実装 | ログイン、一覧、詳細、作成、編集、プロフィール実装 | 5〜8 営業日 |
| テスト、修正 | 主要導線、認証、CRUD の確認と修正 | 2〜4 営業日 |
| リリース準備 | 本番設定、最終確認、デプロイ | 1〜2 営業日 |

## 5. 全体工期の目安

- 最短寄り: 3〜4 週間
- 標準的: 4〜5 週間

## 6. 工期に影響しやすい項目

- 投稿項目数が多いか
- 投稿の状態管理をどこまで入れるか
- パスワード再設定を初回リリースに含めるか
- 一覧に検索、絞り込み、ページングを入れるか

## 7. 補足

- 現在の基本設計レベルでは、工期は比較的読みやすい
- 追加の権限分岐、管理画面、外部連携が入ると別途再見積が必要
