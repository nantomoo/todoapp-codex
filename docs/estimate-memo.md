# 工程別見積メモ

## 1. 前提

- `SPA + Supabase` 構成
- 会員種別は 1 種類
- ファイルアップロード機能なし
- 管理画面なし
- 基本機能は認証、プロフィール、投稿 CRUD
- `demo mode` を併用して単体確認できる
- 1 人開発を想定

## 2. 現在の実装済みスコープ

- 会員登録
- ログイン、ログアウト
- プロフィール参照、更新
- 投稿一覧、投稿詳細
- 投稿作成、更新、削除
- 自分の投稿一覧
- 公開状態変更
- `demo mode` / `supabase mode` 切り替え

## 3. 未実装の拡張候補

- パスワード再設定
- 投稿検索、絞り込み、ページング
- 画像アップロード
- 通知機能
- 管理画面

## 4. ざっくりした段取り

1. 基本設計の確定
2. 詳細設計
3. 開発環境構築
4. Supabase 実装
5. フロント実装
6. テスト、修正
7. リリース準備

## 5. 工程別の目安

| 工程 | 内容 | 目安 |
|---|---|---|
| 基本設計の確定 | 基本設計の確認、修正反映、スコープ確定 | 1〜2 営業日 |
| 詳細設計 | 画面項目、テーブル詳細、RLS 方針の整理 | 2〜4 営業日 |
| 開発環境構築 | SPA 雛形、Supabase 接続、ルーティング初期設定 | 1〜2 営業日 |
| Supabase 実装 | テーブル作成、Migration、Auth、RLS 設定 | 2〜4 営業日 |
| フロント実装 | 認証、一覧、詳細、作成、編集、プロフィール実装 | 5〜8 営業日 |
| テスト、修正 | 主要導線、認証、CRUD の確認と修正 | 2〜4 営業日 |
| リリース準備 | 本番設定、最終確認、デプロイ | 1〜2 営業日 |

## 6. 全体工期の目安

- 現在の実装レベル相当: 3〜4 週間
- 検索、通知、管理画面まで含む拡張版: 4〜6 週間以上

## 7. 工期に影響しやすい項目

- 投稿項目数が多いか
- 投稿の状態管理をどこまで入れるか
- パスワード再設定を初回リリースに含めるか
- 一覧に検索、絞り込み、ページングを入れるか
- 画像アップロードや外部ストレージ連携を入れるか

## 8. 補足

- 現在の実装は静的フロント中心のため、バックエンド独自開発工数は小さい
- 追加の権限分岐、管理画面、外部連携が入ると別途再見積が必要
