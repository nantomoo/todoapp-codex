# 機能一覧

## 1. 認証機能

| 機能ID | 機能名 | 概要 | 利用者 |
|---|---|---|---|
| AUTH-01 | 会員登録 | 表示名、メールアドレス、パスワードで新規登録を行う | ゲスト |
| AUTH-02 | ログイン | 登録済みアカウントでログインする | ゲスト |
| AUTH-03 | ログアウト | 現在のセッションを終了する | 会員 |
| AUTH-04 | セッション維持 | セッション状態を取得し、認証必須画面を保護する | 会員 |
| AUTH-05 | 認証ガード | 未ログインで保護画面へアクセスした場合にログイン画面へ誘導する | システム |

## 2. プロフィール機能

| 機能ID | 機能名 | 概要 | 利用者 |
|---|---|---|---|
| PROF-01 | プロフィール参照 | ログインユーザー自身のプロフィール情報を表示する | 会員 |
| PROF-02 | プロフィール更新 | 表示名と自己紹介を更新する | 会員 |
| PROF-03 | 初期プロフィール生成 | Supabase 上にプロフィールが存在しない場合、初回参照時に作成する | システム |

## 3. 投稿機能

| 機能ID | 機能名 | 概要 | 利用者 |
|---|---|---|---|
| POST-01 | 投稿一覧表示 | 公開中の投稿一覧を表示する | ゲスト、会員 |
| POST-02 | 投稿詳細表示 | 公開中の投稿詳細を表示する。本人は非公開投稿も表示できる | ゲスト、会員 |
| POST-03 | 自分の投稿一覧表示 | ログインユーザー自身の投稿一覧を表示する | 会員 |
| POST-04 | 投稿作成 | 新規投稿を作成する | 会員 |
| POST-05 | 投稿更新 | 自分の投稿を編集する | 会員 |
| POST-06 | 投稿削除 | 自分の投稿を削除する | 会員 |
| POST-07 | 投稿状態変更 | 投稿状態を `draft`、`published`、`archived` に切り替える | 会員 |

## 4. 画面表示、補助機能

| 機能ID | 機能名 | 概要 | 利用者 |
|---|---|---|---|
| UI-01 | モード表示 | 現在の動作モードを `demo mode` または `supabase mode` として表示する | ゲスト、会員 |
| UI-02 | 通知表示 | 成功、警告、エラー通知を画面上に表示し、手動で閉じられる | ゲスト、会員 |
| UI-03 | 404 表示 | 未定義のパスに対して 404 画面を表示する | ゲスト、会員 |
| UI-04 | 入力バリデーション | プロフィール、投稿、ログイン入力に対して必須、文字数、状態値を検証する | システム |
| UI-05 | 送信中制御 | フォーム送信中は入力項目を無効化し、送信ボタン表示を `処理中...` に切り替える | システム |

## 5. データアクセスモード

| 機能ID | 機能名 | 概要 | 利用者 |
|---|---|---|---|
| MODE-01 | Supabase モード | `config.js` に接続情報がある場合、Supabase Auth と DB を利用する | システム |
| MODE-02 | Demo モード | Supabase 未設定時は `localStorage` 上のデモデータで動作する | システム |
| MODE-03 | デモデータ初期化 | `demo mode` 初回起動時にデモ会員、プロフィール、公開投稿を自動投入する | システム |

## 6. 実装上の主要関数

| 関数名 | 役割 |
|---|---|
| `parseRoute` | ハッシュ URL をアプリ内ルートへ変換する |
| `renderApp` | ルートごとの画面描画を統括する |
| `refreshSession` | セッションとプロフィールを再取得して状態を更新する |
| `createDemoStore` | `localStorage` ベースの疑似データストアを生成する |
| `createSupabaseStore` | Supabase ベースのデータストアを生成する |
| `validateProfileInput` | プロフィール入力値を検証する |
| `validatePostInput` | 投稿入力値を検証する |
| `setFormBusy` | フォーム送信中の UI 状態を制御する |
