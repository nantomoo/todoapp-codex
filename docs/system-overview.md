# システム概要

## 1. システム名

会員制投稿アプリ

## 2. 目的

ユーザーがログインし、投稿データの作成・閲覧・更新・削除を行える Web アプリを提供する。バックエンド機能は原則として Supabase に集約し、フロントエンドは SPA として実装する。

## 3. システム構成

```text
[Client]
SPA (React / Vue 想定)

    ↓ HTTPS

[Supabase]
- Auth
- PostgreSQL Database
- Row Level Security
- Edge Functions
```

## 4. 設計方針

- フロントエンドは SPA とし、画面表示とユーザー操作に責務を限定する
- CRUD は可能な限り Supabase Database に直接接続する
- 認可はフロントエンドではなく RLS で担保する
- 秘密情報を扱う処理、外部連携、Webhook 受信は Edge Functions に分離する
- 独自 API サーバーは原則として持たない

## 5. 対象ユーザー

### 5.1 ゲストユーザー

- 未ログインで公開情報のみ閲覧可能

### 5.2 会員ユーザー

- ログイン済みユーザー
- 自分のプロフィール、投稿を管理可能

## 6. 利用想定機能

- 会員登録、ログイン、ログアウト
- プロフィール編集
- 投稿の作成、公開、編集、削除

## 7. 非機能要件

- 認証必須機能は未ログイン状態で利用不可
- 権限制御は業務テーブルに対する RLS で強制する
- 主要画面は PC とスマートフォンに対応する
- 重要操作は監査ログとして保存する
- データベース変更はマイグレーションで管理する

## 8. 補足

- 公開系のデータ取得は Supabase の RLS を前提とする
- 秘密鍵を必要とする外部連携が発生する場合は Edge Functions 経由で実行する
